Усложнением 6ой миссии серии является то, что теперь для освещения помещения может понадобиться более одной лампочки. И это 5й аргумент функции - сколько лампочек нужно для освещения помещения.

Например, если для освещения помещения нужно 3 лампочки, то мы не засчитываем то время, когда лампочек было всего 2 или только одна. Если последний аргумент функции не передан, значит достаточно одной лампочки для освещения помещения.

Задача все та же - узнать сколько времени помещение было освещено (в этой задаче можно сказать - достаточно освещено).

Input: Пять аргументов и только первый обязательный. Первый – {% if is_js %}an array of Date objects (вместо Date может быть [Date, number]){% else %}a list of datetime objects(вместо datetime может быть [datetime, int]){% endif %}, а второй и третий – {% if is_js %}the Date objects{% else %}the datetime objects{% endif %}. Четвертый аргумент (operating) - {% if is_js %}integer{% else %}timedelta object{% endif %} - как долго работала лампочка. Пятый аргумент - положительный int

Output: Количество секунд как integer.

Example:

{% if is_js %} 
sumTwo(2, 3) == 5
sumTwo(5, 7) == 12
{% else %} 
sum_light([
    (datetime(2015, 1, 12, 10, 0, 10), 3),
    datetime(2015, 1, 12, 10, 0, 20),
    (datetime(2015, 1, 12, 10, 0, 30), 3),
    (datetime(2015, 1, 12, 10, 0, 30), 2),
    datetime(2015, 1, 12, 10, 0, 40),
    (datetime(2015, 1, 12, 10, 0, 50), 2),
], req=2) == 20

sum_light([
    (datetime(2015, 1, 12, 10, 0, 10), 3),
    datetime(2015, 1, 12, 10, 0, 20),
    (datetime(2015, 1, 12, 10, 0, 30), 3),
    (datetime(2015, 1, 12, 10, 0, 30), 2),
    datetime(2015, 1, 12, 10, 0, 40),
    (datetime(2015, 1, 12, 10, 0, 50), 2),
], req=3) == 0

sum_light([
    (datetime(2015, 1, 12, 10, 0, 10), 3),
    datetime(2015, 1, 12, 10, 0, 20),
    (datetime(2015, 1, 12, 10, 0, 50), 3),
    (datetime(2015, 1, 12, 10, 0, 30), 2),
    datetime(2015, 1, 12, 10, 0, 40),
    (datetime(2015, 1, 12, 10, 0, 50), 2),
], req=3) == 10
{% endif %}

Предусловия:

  • Массив нажатий на кнопку всегда отсортирован по возрастанию.
  • В массиве нажатий на кнопку нет повторяющихся элементов.
  • Количество элементов всегда четное число (это значит, что лампочка, в конце концов, будет выключена).
  • Минимально возможная дата 1970-01-01.
  • req > 0
